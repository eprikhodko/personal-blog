Когда мы только начинаем учиться писать код, мы фокусируемся только на одной вещи: чтобы наш код работал. Если работает - супер. Если не работает, надо сделать так, чтобы работал. Все. Больше от нас ничего не требуется.

Когда у нас появляется больше опыта, кроме того, чтобы наш код работал, было бы хорошо, чтобы он был бы еще и _качественным_. У одной и той же задачи может быть два и более возможных решения, но одно из этих решений может быть _лучше_ чем другое.

Как можно оценить качество кода? Одно из качеств хорошего кода - это его поддерживаемость. Поддерживаемость кода включает в себя такие вещи как читабельность кода, организация кода, и модульность кода.

Однако, существует еще один признак качественного кода, и это его _эффективность_. К примеру, у нас может быть два фрагмента кода, оба из которых достигают одной и той же цели, но один _выполняется быстрее чем другой_.

Вот две функции, которые выводят четные числа от 2 до 100:

```py
def print_numbers_version_one():
number = 2
while number <= 100:
# If number is even, print it:
if number % 2 == 0:
print(number)
number += 1
```

```py
def print_numbers_version_two():
number = 2
while number <= 100:
print(number)
# Increase number by 2, which, by definition,
# is the next even number:
number += 2
```

Как думаете, какая из этих функций работает быстрее?

Вторая версия быстрее, потому что она прогоняет цикл только 50 раз, а первая - 100 раз.

Мы должны стараться писать _эффективный и быстрый_ код. Чтобы понять, как писать такой код, мы должны понять, что такое структуры данных, и как разные структуры данных могут влиять на скорость выполнения нашего кода.
